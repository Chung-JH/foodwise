# docs/idea.md

## 为什么做这个项目？

### 问题观察
经常点外卖的人面临几个共性痛点：
1. **选择困难**：打开外卖 App 面对几百家店几千道菜，每天花 10 分钟纠结吃什么
2. **无意识重复**：回顾发现连续一周都在吃炸鸡汉堡，营养严重失衡
3. **备注重复劳动**：每次点餐都要手打"不要香菜、不要葱、少油"
4. **帮别人点记不住**：帮室友、家人点餐时记不住对方的忌口和口味
5. **预算与营养难兼顾**：想省钱又想吃好，缺少智能平衡建议

### 为什么用大模型？
- 传统做法是勾选标签、查营养表、写 if-else 规则 —— 不灵活、维护成本高
- 大模型可以：
  - 从自由文本中提取结构化偏好（"喜欢川菜但不能太辣" → 标签化）
  - 从菜品照片中识别菜名、食材、营养信息（多模态理解）
  - 综合用户画像 + 饮食历史 + 预算约束做推理推荐（比规则更灵活）
  - 根据菜品类型生成自然语言备注（比字符串拼接更通顺）

### 为什么是 MVP？
- 这是课程设计项目，时间有限，重在跑通核心链路
- 不追求接入真实外卖平台，不做支付、不做云部署
- 用 Mock 模式保证无 API 时也能演示
- 重点展示"LLM 驱动"的差异化设计思路

### 核心假设
- 用户愿意花 30 秒记录一餐（文本或拍照），换取更好的下一餐推荐
- LLM 的营养估算虽不精确，但作为粗略参考已有实用价值
- 菜品库通过用户上传不断增长，越用越准

### 一句话总结
用多模态大模型帮点外卖的人解决"吃什么"的决策疲劳，同时兼顾营养均衡和预算控制。
